My main "creative insight" in this project was to try and order things and explain them so that they would be accessible to beginners. The underlying mathematical concepts I'm explaining were developed by other people.

That said, my code largle values pedagogical explanation over efficiency, so if you want your game to run fast, you actually wouldn't want to copy and paste my code. Below, I provide a list that will be updated as I go of resources, books, thoughts on said resources, and so forth, with priority given to those whose material is accessible to total beginners. If you're thinking "how did you not mention [famous person here]?" it's because this is for an audience of true beginners.

I initially used ChatGPT and Gemini but later settled on Claude for many of my basic coding questions. ONLINE REFERENCES/CREDITS SPECIFIC TO RAYMARCHING, GODOT, ART, or TECH ART
----------

Freya Holmér - She's got a twelve hour intro series to shaders on YouTube which she could justifiably charge a zillion dollars for but for some people the work is enough. Shader code is similar across engines, so don't be dissuaded that it's in Unity if you're working in another engine.  

GDQuest - For learning the Godot engine and the basic programming knowledge needed to drive it. I paid for their online class and it's worth it. They don't cover the creation of artistic assets, but ironically, provide great ones so you can focus on learning the game dev side before falling down the Blender rabbit hole. I'll be using their Gobot model but they have more. 

CGCookie - This was totally worth the monthly membership to learn Blender. The library on designing game assets is more limited than I'd like, but there's a big gulf awaiting you after the online Blender tutorial about making a donut, and as useful as the donut tutorial is, you'll have to go deeper in Blender. But do make the donut.

Íñigo Quílez - Although he's famous for raymarching, I would actually advise against starting with IQ if you're new to tech art, Godot, or game design, as his approach is intermediate/advanced and he'll melt your brain with his proofs before you've had time to deal with more basic stuff, like how shaders work. Or how to get a project and some nodes going in Godot. (I would read Penny de Byl's book first, for example.) You should, however, refer to his simple list of 3D SDF functions for my tutorial, from which I explicitly draw, which is early on. You might see specific citations to his word in the code instead of here. Due to his omnipresence in pioneering the field, it sometimes feels more like a question of what shouldn't be attributed to him rather than what should.
https://iquilezles.org/articles/distfunctions/

Programming with Mosh - Although his YouTube class about Python is about Python, GDScript is very similar to Python, so I'm counting it. If you have zero programming knowledge at all, and want to have a meaningful sense of it before jumping into a game engine, I'd do this first.

CodeBeauty - Saldina Nurak kind of reminds me of the ASMR voices I listen to at bedtime, which is really refreshing in a sea of programming talking heads that somehow feel too loud for my brain. She takes a language I used to find scary (C++) and makes it not scary. As far as compliments to teachers go, that's a really high one. Although you don't technically need C++ itself, the shader language you're working in, whether in Godot or elsewhere, is likely styled the same way, and if the style is totally unfamiliar to you, you'll need to learn the style somewhere via an intro C++ series.


BOOK RECOMMENDATIONS
--------------------

Mathematics for Game Programming and Computer Graphics, by Penny de Byl - This is the best book I've yet to read so far about any of this, and I've read about 20+ of them. She uses Python to keep things simple while she takes you through a lot of the essential math behind game design and computer graphics. If you're looking for a textbook because you can't go to a university, this is it. You'll bounce over to Godot tutorials to keep you focused on the dream, but I constantly come back to this book to remember things I keep forgetting. A phenomenal work in teaching.

Godot 4 Game Development Projects, by Chris Bradfield - Tiny little games to make in Godot that introduce you, one chunk at a time, to important concepts. One of the best pieces of advice I got was to make tiny games to learn, and this gets you into the important philosophy taught by my ceramics teacher, "no work is too precious". She was referring to clay pieces that often exploded in the kiln, but it applies to game design.

Game Development with Blender and Godot, by Kumsal Obuz - This takes you through creating a 3D game level, start to finish. Although it's written for someone who's never touched Godot or Blender, it hits everything from baking in pathfinding to adding a water shader. If you choose to follow the tutorial word for word, do yourself a favor and download the version of Godot it was written for, so you don't go nuts. It's easier to learn the updates to Godot later, after you've learned the basics, and I wouldn't call it sufficiently educational to step every half hour to diagnose an error caused by a version change.
